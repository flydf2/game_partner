# API 请求重试机制调整

## 修改时间
2026-03-25

## 修改内容
调整 API 请求的重试策略，所有请求异常不再进行重试。

## 修改文件
- `src/api/index.js`

## 修改详情

### 修改前
`shouldRetry` 函数会对以下错误类型进行重试：
- 网络错误（无响应）
- 服务器错误（5xx 状态码）
- 429 Too Many Requests 错误

重试配置：
- 最大重试次数：3 次
- 初始延迟：1000ms
- 最大延迟：10000ms
- 退避策略：指数退避

### 修改后
`shouldRetry` 函数直接返回 `false`，所有请求异常都不进行重试。

## 修改原因
根据业务需求，接口请求异常不需要重试，避免重复请求对服务器造成压力，同时让用户能够更快地看到错误提示。

## 影响范围
所有使用 `withRetry` 函数包装的 API 调用，包括但不限于：
- 专家详情相关 API
- 用户相关 API
- 通知相关 API
- 消息相关 API
- 订单相关 API
- 游戏相关 API
- 社区相关 API
- 陪玩相关 API

## 注意事项
- 用户在遇到网络错误或服务器错误时，需要手动点击重试按钮
- 建议在 UI 层面提供友好的错误提示和重试机制
- 对于关键操作，可以考虑在业务层面实现更精细的错误处理策略

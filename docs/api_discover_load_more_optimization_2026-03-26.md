# 发现页面加载更多逻辑优化

## 更新时间
2026-03-26

## 问题描述
在发现页面（Discover.vue）中，加载更多陪玩的逻辑存在问题：
- 当滚动到底部时，会先增加页码，然后再调用加载函数
- 这可能导致页码管理混乱，特别是在网络请求延迟的情况下
- 影响用户体验，可能出现重复加载或加载失败的情况

## 解决方案
1. **优化页码管理**：在loadMoreData函数内部计算下一页页码，而不是在滚动事件中提前增加页码
2. **简化滚动事件处理**：移除滚动事件中的页码增加操作，让loadMoreData函数负责计算和传递正确的页码
3. **保持状态一致性**：确保页码的增加与数据加载操作在同一函数中，保证状态的一致性

## 技术细节

### 代码修改
- 修改了`loadMoreData`函数，在函数内部计算下一页页码：`const nextPage = playmateStore.currentPage + 1`
- 将计算出的`nextPage`作为参数传递给`fetchPlaymates`函数
- 移除了`throttledHandleScroll`函数中的`playmateStore.incrementPage()`调用

### 实现逻辑
1. 滚动事件触发时，检查是否需要加载更多数据
2. 如果需要，调用`loadMoreData`函数
3. `loadMoreData`函数计算下一页页码并发起请求
4. 请求成功后，`fetchPlaymates`函数会更新store中的页码和hasMore状态
5. 这样确保了页码的增加与数据加载操作的原子性

## 影响范围
- 仅影响发现页面的加载更多逻辑
- 不影响其他页面或功能

## 测试建议
1. 访问发现页面，滚动到底部，检查是否能正确加载更多数据
2. 测试多次滚动到底部的情况，确保不会重复加载或出现错误
3. 测试筛选条件或排序方式变更后，加载更多功能是否正常
4. 测试网络延迟情况下的加载行为

## 预期效果
- 加载更多数据时，页码管理更加准确
- 避免了因网络延迟导致的页码混乱问题
- 提升了用户体验，加载过程更加流畅
- 减少了加载失败的可能性
